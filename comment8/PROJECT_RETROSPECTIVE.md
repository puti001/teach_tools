## 📝 專案復盤紀錄

### 【2026-05-26】Puti-AI 學生期末評語提示詞生成器 (V8 升級 V9)

* **專案主題**：學生評語批次提示詞生成與歷史快取、多主題管理系統。
* **對話摘要**：
  - 修正原先 V8 版本中生成提示詞會無限重複 push 的 bug，並將單一學生卡片簡化（移除贅餘的特質顯示，僅保留復原按鈕）。
  - 將零散的個別提示詞，改進為「全班一次性合併」的批次提示詞結構，省去老師複製 30 次的工作。
  - 新增「本機歷史紀錄管理」，支援自動以 民國年月+年級 命名與重名流水號。
  - 新增備份匯出與匯入功能（`.json`），並優化為匯入後自動選取並載入。
  - 引入 HSL 多主題切換（5 色）與網頁內建的「使用教學」分頁，並將圖片搬移至專案目錄進行相對路徑優化。
  
* **🎓 習得關鍵技能**：
  1. **Token 節省型合併 Prompt 設計**：用公共前綴 + 結構化學生特質 `學號.姓名{特質1、特質2}` 代替多個重複 Prompt。
  2. **`contenteditable` 雙向綁定機制**：點擊網頁文字直接編輯，並在 `onblur` 時自動解析與存檔。
  3. **自動命名與重名流水號演算法**：計算民國年並在名稱重複時以 `while` 遞增 `_1`、`_2`。
  4. **GitHub 憑證自動部署**：利用 Windows 終端機與本地腳本配合 PAT 進行一鍵推送，免手動操作。

* **💡 重要提示詞與模板結構**：
  * **合併用 Prompt 模板**：
    ```text
    請為以下{grade}年級的學生，以正向、溫暖、鼓勵的語氣，根據各學生特質，生成{wordCount}字的期末評語：
    01.馬建宏{聰穎靈活、自律自覺、待人和善、語文能力強}
    02.范建宏{理解力強、誠實守分、人緣良好、寫作天賦高}
    ```

* **❤️ 開發心得與備忘**：
  - **使用者體驗 (UX) 的直覺性**：匯入備份後如果畫面沒變，使用者會感到困惑。改為「匯入即自動載入第一筆」後，操作感流暢許多。
  - **Token 最佳化**：對於需要大量批次處理的 AI 網頁工具，把重複的 prompt 範本合併成單一指令，不僅是幫使用者省時間，更是在幫使用者的錢包（AI 額度）省錢。

### 【2026-05-27】快速分類功能整合與卡片自訂系統 (V9.1 升級)

* **專案主題**：整合舊版拖曳分類功能、行動端點擊移動、卡片標籤雙擊編輯與自動化部署優化。
* **對話摘要**：
  - 新增「快速分類」分頁，將舊版（V6）的 8 型分類區與拖放機制重新寫入 V9，並與 HSL 配色主題完美融合。
  - 頁籤命名優化：`[個別生成]` ➡️ `[個別生成]`（原本為生成器），`[拖曳分類生成器]` ➡️ `[快速分類]`。
  - **行動端點擊移動**：為解決觸控螢幕不支援 HTML5 拖放的問題，實作了「點選待分類學生（高亮呼吸燈），再點選分類區即可移入」的點擊移動機制。在分類區中點選學生則自動移回。
  - **雙擊即時編輯**：分類卡片的「標題」與「四字評語」支援雙擊編輯。變更分類標題後，系統會自動在資料庫與已分類學生中更新對應，無縫連動。
  - **自動備份與重置**：卡片自訂配置即時存於 `localStorage`。點擊「重置分類」時會一併還原卡片為預設配置。
  
* **🎓 習得關鍵技能**：
  1. **行動端拖放 Fallback（點擊移動）**：在桌面端保留 `drag & drop` 的同時，利用 `click` 事件與選取狀態組（Set）在行動端達成無縫的元素位置移轉。
  2. **動態 contenteditable 連動更新**：當 DOM element 作為 key 或關聯索引被修改時，同步更新 JavaScript 的 State（如 `dragClassification`）與 LocalStorage。
  3. **自動化部署隔離（遞迴防禦）**：執行自動拷貝腳本時，必須將克隆的臨時路徑（`temp_clone`）設定在專案目錄外部（如 AppData），否則遞迴拷貝會複製自己引發硬碟爆滿。
  4. **Git 無人值守部署最佳化**：使用 `--depth 1` 做快速克隆以應對大 Repository；使用 `-c credential.helper=` 與 `GIT_TERMINAL_PROMPT=0` 避免在背景執行時卡在憑證彈窗。
